# CareerLog 요구사항 정의서

> CareerLog 1차 MVP 기준의 기능 및 비기능 요구사항을 정의한 문서입니다.  
> 개발 진행에 따라 요구사항, 우선순위와 구현 상태는 변경될 수 있습니다.
>
> 최종 수정일: 2026-07-19

## 1. 문서 개요

### 1.1 프로젝트 목적

CareerLog는 취업 준비 과정에서 발생하는 지원 현황, 면접 일정, 면접 질문과 답변 회고를 한곳에서 관리하는 개인 취업 관리 플랫폼입니다.

단순히 지원 이력을 저장하는 데 그치지 않고, 면접에서 부족했던 답변을 기록하고 개선 답변과 복습 대상을 관리하여 다음 면접 준비까지 연결하는 것을 목표로 합니다.

### 1.2 핵심 사용자 흐름

```text
회원가입 및 로그인
→ 지원 건 등록
→ 지원 상태와 다음 행동 관리
→ 지원 건에 면접 기록 등록
→ 면접 질문과 당시 답변 기록
→ 답변 점수와 부족했던 점 평가
→ 개선 답변 작성
→ 복습 필요 질문 관리
→ 다음 면접 준비
```

### 1.3 대상 사용자

- 취업 지원 현황을 체계적으로 관리하려는 구직자
- 면접에서 받은 질문과 답변을 기록하고 싶은 사용자
- 반복해서 부족한 기술 개념과 답변을 복습하려는 사용자

### 1.4 용어 정의

| 용어 | 설명 |
|---|---|
| 사용자 | CareerLog에 가입하고 개인 취업 데이터를 관리하는 사람 |
| 지원 건 | 특정 회사와 직무 또는 채용 공고에 대한 관심·지원 기록 |
| 면접 | 하나의 지원 건에서 진행되는 전화, 화상, 기술, 인성, 최종 면접 등의 기록 |
| 면접 질문 | 면접에서 받은 개별 질문과 당시 답변, 자기평가 및 개선 답변을 포함하는 기록 |
| 복습 대상 | 사용자가 `needReview`를 활성화하여 다시 확인하도록 지정한 면접 질문 |
| 소유권 | 특정 데이터가 현재 로그인한 사용자에게 귀속되는지 여부 |

## 2. 범위

### 2.1 1차 MVP 목표

> 로그인한 사용자가 자신의 지원 건을 관리하고, 각 지원 건의 면접 질문과 답변 회고를 기록하며, 복습이 필요한 질문을 다시 조회할 수 있다.

### 2.2 MVP 포함 범위

- 회원가입 및 로그인
- JWT 기반 인증
- Access Token 재발급과 로그아웃
- 내 정보 조회
- 지원 건 등록·목록·상세·수정·삭제
- 지원 상태 변경
- 지원 건별 면접 등록·목록·상세·수정·삭제
- 면접별 질문 등록·목록·상세·수정·삭제
- 답변 점수, 부족했던 점, 개선 답변 기록
- 복습 필요 여부 변경
- 복습 필요 질문 목록 조회
- 사용자별 데이터 접근 제어
- 입력값 검증과 일관된 오류 응답

### 2.3 MVP 제외 범위

- AI 기반 답변 자동 평가
- 채용 공고 크롤링
- 이력서·포트폴리오 파일 첨부
- 이메일 또는 메시지 발송
- 외부 캘린더 연동
- 팀 또는 조직 단위 협업
- 복잡한 통계와 분석 차트
- 회사 및 채용 공고 별도 엔티티 분리
- n8n 자동화
- Temporal 워크플로
- Oracle Cloud 배포
- 완전한 모바일 최적화

## 3. 요구사항 상태 기준

| 상태 | 설명 |
|---|---|
| 완료 | 현재 코드에 구현되어 정상 동작을 확인한 요구사항 |
| 일부 구현 | 핵심 기능 일부만 구현되었거나 인증·예외 처리 등 보완이 필요한 요구사항 |
| 설계 | 요구사항과 데이터 구조가 정의되었으나 아직 코드로 구현하지 않은 항목 |
| 예정 | MVP에 포함되지만 아직 상세 설계 또는 구현하지 않은 요구사항 |
| 제외 | 현재 MVP에서는 구현하지 않는 요구사항 |

우선순위는 다음 기준을 사용합니다.

| 우선순위 | 설명 |
|---|---|
| 필수 | MVP 완료를 위해 반드시 필요한 요구사항 |
| 중요 | 핵심 사용 흐름을 보완하며 가급적 MVP에 포함할 요구사항 |
| 선택 | MVP 이후 또는 일정에 따라 추가할 수 있는 요구사항 |

## 4. 기능 요구사항

## 4.1 사용자 및 인증

| ID | 요구사항 | 우선순위 | 상태 | 관련 테이블 |
|---|---|---|---|---|
| `AUTH-001` | 사용자는 이메일, 비밀번호, 닉네임을 입력하여 회원가입할 수 있어야 한다. | 필수 | 예정 | `users` |
| `AUTH-002` | 회원가입 시 이메일은 중복될 수 없어야 한다. | 필수 | 예정 | `users` |
| `AUTH-003` | 비밀번호는 평문이 아닌 단방향 해시 형태로 저장되어야 한다. | 필수 | 예정 | `users` |
| `AUTH-004` | 사용자는 이메일과 비밀번호로 로그인할 수 있어야 한다. | 필수 | 예정 | `users` |
| `AUTH-005` | 로그인 성공 시 Access Token과 Refresh Token을 발급해야 한다. | 필수 | 예정 | `refresh_tokens` |
| `AUTH-006` | 유효한 Refresh Token으로 Access Token을 재발급할 수 있어야 한다. | 필수 | 예정 | `refresh_tokens` |
| `AUTH-007` | 사용자는 로그아웃할 수 있어야 하며, 해당 Refresh Token은 재사용할 수 없어야 한다. | 필수 | 예정 | `refresh_tokens` |
| `AUTH-008` | 인증된 사용자는 자신의 기본 정보를 조회할 수 있어야 한다. | 중요 | 예정 | `users` |
| `AUTH-009` | 인증이 필요한 기능은 유효한 Access Token이 없으면 사용할 수 없어야 한다. | 필수 | 예정 | - |
| `AUTH-010` | 만료되거나 위조된 토큰으로 요청하면 인증 오류를 반환해야 한다. | 필수 | 예정 | - |

### 회원가입 입력 규칙

| 항목 | 규칙 |
|---|---|
| 이메일 | 필수, 이메일 형식, 최대 255자, 중복 불가 |
| 비밀번호 | 필수, 최소 길이와 조합 규칙은 인증 구현 시 확정 |
| 닉네임 | 필수, 최대 50자 |

### 면접 입력 규칙

- 면접 유형과 면접 일시는 필수입니다.
- 면접관 수는 0 이상의 정수만 허용합니다.
- 난이도는 적용 시 1점부터 5점 범위로 제한합니다.
- 면접은 반드시 현재 사용자가 소유한 지원 건에 연결되어야 합니다.

## 9. 인수 기준

다음 조건을 모두 충족하면 CareerLog 1차 MVP가 완료된 것으로 판단합니다.

- 사용자가 회원가입하고 로그인할 수 있습니다.
- 로그인 응답으로 JWT를 발급받고 인증 API를 호출할 수 있습니다.
- 사용자가 자신의 지원 건을 등록·조회·수정·삭제할 수 있습니다.
- 다른 사용자의 지원 건에는 접근할 수 없습니다.
- 하나의 지원 건에 여러 면접을 등록·조회·수정·삭제할 수 있습니다.
- 하나의 면접에 여러 질문과 답변 회고를 등록할 수 있습니다.
- 질문별 답변 점수, 부족했던 점과 개선 답변을 저장할 수 있습니다.
- 질문의 복습 필요 여부를 변경할 수 있습니다.
- 복습이 필요한 자신의 질문만 별도로 조회할 수 있습니다.
- 입력값 오류와 존재하지 않는 데이터에 대해 일관된 오류 응답을 반환합니다.
- 공개 저장소에 실제 비밀번호, JWT Secret과 데이터베이스 접속 비밀번호가 포함되지 않습니다.
- 프론트엔드에서 위 핵심 기능을 사용할 수 있습니다.
